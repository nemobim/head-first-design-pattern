# 프록시 패턴

## 개념

프록시 패턴은 특정 객체로의 접근을 제어하는 대리인(대변자)을 제공하는 패턴이다.  
프록시는 실제 객체를 대신하여 클라이언트의 요청을 받고, 필요에 따라 실제 객체에게 작업을 위임한다.

핵심은 클라이언트와 실제 객체 사이에 **프록시 객체**를 두어 접근 제어, 지연 로딩, 원격 접근 등의 부가 기능을 제공하는 것이다.

## 구조

**Subject 인터페이스**

프록시와 실제 객체가 구현하는 공통 인터페이스

**RealSubject**

실제 작업을 처리하는 객체

**Proxy**

RealSubject를 참조하며, 클라이언트 요청을 제어하거나 위임

```txt
Client
  └─ proxy.request()

Proxy
  ├─ realSubject: RealSubject
  └─ request() → realSubject.request()

RealSubject
  └─ request() // 실제 작업 수행
```

> 프록시는 실제 객체와 동일한 인터페이스를 구현하여, 클라이언트가 프록시인지 실제 객체인지 구분하지 못하게 한다.

## 프록시 패턴의 종류

### 원격 프록시 (Remote Proxy)

다른 JVM이나 네트워크상의 객체를 로컬에서 사용할 수 있게 하는 대리인이다.  
원격 객체의 메서드를 호출하면 프록시가 네트워크 통신을 처리한다.

**특징**

- 원격 객체: 다른 주소 공간에서 실행되는 객체
- 로컬 대변자: 원격 객체에 메서드 호출을 전달하는 프록시
- 네트워크 통신 관련 저수준 작업을 프록시가 처리

### 가상 프록시 (Virtual Proxy)

생성 비용이 큰 객체의 생성을 지연(lazy loading)시키는 프록시다.  
실제 객체가 필요한 시점까지 생성을 미루고, 그 전까지는 프록시가 대신한다.

**특징**

- 객체 생성 전이나 생성 중에 프록시가 대신 동작
- 객체 생성이 완료되면 실제 객체에 요청을 위임
- 무거운 리소스(이미지, 대용량 데이터)의 로딩 최적화에 유용

### 보호 프록시 (Protection Proxy)

객체에 대한 접근 권한을 제어하는 프록시다.  
클라이언트의 권한에 따라 특정 메서드 호출을 허용하거나 거부한다.

### 그 외

- **캐싱 프록시**: 결과를 캐싱하여 성능 향상
- **스마트 레퍼런스**: 참조 횟수 카운팅, 로깅 등 부가 기능 제공

## 예시 (원격 프록시 개념)

뽑기 기계를 원격으로 모니터링하는 시스템

### 실제 객체 (RealSubject)

```ts
class GumballMachine {
	location: string;
	count: number;
	state: string;

	constructor(location: string, count: number) {
		this.location = location;
		this.count = count;
		this.state = 'sold out';
	}

	getLocation(): string {
		return this.location;
	}

	getCount(): number {
		return this.count;
	}

	getState(): string {
		return this.state;
	}
}
```

### 모니터 (Client)

```ts
class GumballMonitor {
	machine: GumballMachine; // 원격 프록시를 주입받음

	constructor(machine: GumballMachine) {
		this.machine = machine;
	}

	report() {
		console.log(`뽑기 기계 위치: ${this.machine.getLocation()}`);
		console.log(`현재 재고: ${this.machine.getCount()} 개`);
		console.log(`현재 상태: ${this.machine.getState()}`);
	}
}
```

### 사용

```ts
const gumballMachine = new GumballMachine('서울', 100);
const monitor = new GumballMonitor(gumballMachine);
monitor.report();
// 실제로는 네트워크를 통해 원격 객체의 정보를 가져옴
```

> 실제 구현에서는 `GumballMachine` 대신 **원격 프록시**를 주입하여 네트워크 통신을 처리한다.

## 예시 (가상 프록시)

이미지 로딩을 지연시키는 ImageProxy

### Subject 인터페이스

```ts
interface Icon {
	getIconWidth(): number;
	getIconHeight(): number;
	printIcon(): void;
}
```

### Proxy

```ts
class ImageProxy implements Icon {
	imageIcon: Icon | null = null;
	imageURL: string;
	retrieving: boolean = false;

	constructor(url: URL) {
		this.imageURL = url.toString();
	}

	getIconWidth(): number {
		if (this.imageIcon !== null) {
			return this.imageIcon.getIconWidth();
		} else {
			return 800; // 기본 너비 반환
		}
	}

	getIconHeight(): number {
		if (this.imageIcon !== null) {
			return this.imageIcon.getIconHeight();
		} else {
			return 600; // 기본 높이 반환
		}
	}

	printIcon(): void {
		if (this.imageIcon !== null) {
			// 이미지 로딩이 완료된 경우
			this.imageIcon.printIcon();
		} else {
			// 이미지 로딩 중
			console.log('이미지를 불러오는 중...');
			if (!this.retrieving) {
				this.retrieving = true;
				// 백그라운드에서 실제 이미지 로딩 시작
				this.loadImage();
			}
		}
	}

	private loadImage(): void {
		// 실제 이미지를 비동기로 로딩
		// 로딩 완료 후 this.imageIcon에 할당
	}
}
```

### 장점

- 이미지가 필요한 시점까지 로딩을 지연시켜 초기 로딩 시간 단축
- 로딩 중에도 기본 크기 정보를 제공하여 레이아웃 깨짐 방지
- 실제 이미지 객체 생성을 프록시가 제어

## 원격 서비스 만들기 (RMI 기반)

1. **원격 인터페이스 만들기**
   - 클라이언트가 원격으로 호출할 메서드 정의
   - 스텁(프록시)과 실제 서비스가 이 인터페이스를 구현

2. **서비스 구현 클래스 만들기**
   - 실제 작업을 처리하는 클래스

3. **RMI 레지스트리 실행하기**
   - 원격 서비스를 등록할 레지스트리 시작

4. **원격 서비스 실행하기**
   - 서비스 인스턴스를 생성하고 레지스트리에 등록

## 프록시 패턴 vs 데코레이터 패턴

두 패턴은 구조가 유사하지만 의도가 다르다.

**프록시 패턴**

- 접근 제어가 목적
- 프록시가 객체 생성, 접근 권한, 원격 통신 등을 제어
- 클라이언트는 프록시의 존재를 인식하지 못함

**데코레이터 패턴**

- 기능 추가가 목적
- 런타임에 동적으로 객체에 새로운 책임을 추가
- 클라이언트가 의도적으로 데코레이터를 감싸서 사용

> 프록시는 "대리인"이고, 데코레이터는 "포장지"다.

## 언제 사용하면 좋은가?

**✔ 원격 객체 접근이 필요할 때 (원격 프록시)**

네트워크 통신, RPC, 마이크로서비스 간 통신

**✔ 객체 생성 비용이 클 때 (가상 프록시)**

이미지 로딩, 대용량 파일, 복잡한 초기화 작업

**✔ 접근 권한을 제어해야 할 때 (보호 프록시)**

관리자/일반 사용자 구분, 읽기/쓰기 권한 관리

**✔ 부가 기능이 필요할 때**

로깅, 캐싱, 지연 실행, 참조 카운팅

## 프론트엔드 활용 예시

### 이미지 레이지 로딩 (가상 프록시)

```ts
class ImageLoader {
	private cache = new Map<string, HTMLImageElement>();

	loadImage(url: string): Promise<HTMLImageElement> {
		// 캐시에 있으면 즉시 반환
		if (this.cache.has(url)) {
			return Promise.resolve(this.cache.get(url)!);
		}

		// 없으면 로딩 후 캐시에 저장
		return new Promise((resolve, reject) => {
			const img = new Image();
			img.onload = () => {
				this.cache.set(url, img);
				resolve(img);
			};
			img.onerror = reject;
			img.src = url;
		});
	}
}

// 사용
const loader = new ImageLoader();
loader.loadImage('/heavy-image.jpg').then((img) => {
	document.body.appendChild(img);
});
```

### API 프록시 (원격 프록시 + 캐싱)

```ts
class APIProxy {
	private cache = new Map<string, any>();
	private pending = new Map<string, Promise<any>>();

	async fetch(endpoint: string): Promise<any> {
		// 캐시 확인
		if (this.cache.has(endpoint)) {
			console.log('캐시에서 반환');
			return this.cache.get(endpoint);
		}

		// 중복 요청 방지
		if (this.pending.has(endpoint)) {
			console.log('진행 중인 요청 대기');
			return this.pending.get(endpoint);
		}

		// 새 요청
		const promise = fetch(endpoint)
			.then((res) => res.json())
			.then((data) => {
				this.cache.set(endpoint, data);
				this.pending.delete(endpoint);
				return data;
			});

		this.pending.set(endpoint, promise);
		return promise;
	}
}

// 사용
const api = new APIProxy();
api.fetch('/api/users'); // 실제 요청
api.fetch('/api/users'); // 캐시 반환
```

### React의 lazy와 Suspense (가상 프록시)

```tsx
import { lazy, Suspense } from 'react';

// 컴포넌트를 동적으로 import (가상 프록시)
const HeavyComponent = lazy(() => import('./HeavyComponent'));

function App() {
	return (
		<Suspense fallback={<div>로딩 중...</div>}>
			<HeavyComponent />
		</Suspense>
	);
}
```

`lazy`가 프록시 역할을 하여, 컴포넌트가 실제로 필요한 시점에 로드한다.

### Proxy 객체를 이용한 접근 제어 (보호 프록시)

```ts
interface User {
	name: string;
	email: string;
	password: string;
	role: 'admin' | 'user';
}

function createUserProxy(user: User, currentUserRole: 'admin' | 'user'): User {
	return new Proxy(user, {
		get(target, prop) {
			// password는 본인 또는 관리자만 접근 가능
			if (prop === 'password' && currentUserRole !== 'admin') {
				throw new Error('접근 권한이 없습니다.');
			}
			return target[prop as keyof User];
		},
		set(target, prop, value) {
			// role 변경은 관리자만 가능
			if (prop === 'role' && currentUserRole !== 'admin') {
				throw new Error('권한 수정 불가');
			}
			target[prop as keyof User] = value;
			return true;
		},
	});
}

// 사용
const user: User = { name: '홍길동', email: 'hong@example.com', password: '1234', role: 'user' };
const userProxy = createUserProxy(user, 'user');

console.log(userProxy.name); // "홍길동"
console.log(userProxy.password); // Error: 접근 권한이 없습니다.
```

JavaScript의 `Proxy` 객체를 사용하면 프록시 패턴을 간단히 구현할 수 있다.

## 정리

프록시 패턴은 실제 객체로의 **접근을 제어**하는 것이 핵심이다.

- **원격 프록시**: 네트워크 통신 처리
- **가상 프록시**: 지연 로딩으로 성능 최적화
- **보호 프록시**: 접근 권한 제어

프론트엔드에서는 이미지 레이지 로딩, API 캐싱, 컴포넌트 코드 스플리팅, 접근 제어 등에 활용된다.
