```java
public interface Iterator {
  boolean hasNext();
  MenuItem next();
}
```

- DinnerMenu 클래스에 사용할 구상 Iterator 클래스 만들기

```java
public class DinnerMenuIterator implements Iterator {
  MenuItem[] items;
  int position = 0;

  public DinnerMenuIterator(MenuItem[] items) {
    this.items = items;

  }

  public MenuItem next() {
    MenuItem menuItem = items[position];
    position = position + 1;
    return menuItem;
  }

  public boolean hasNext() {
    if (position >= items.length || items[position] == null ) {
      return false;
    } else {
      return true;
    }
  }
}
```

# 반복자 패턴의 정의

- 반복자 패턴: 컬렉션의 구현 방법을 노출하지 않으면서 집합체 내의 모든 항목에 접근하는 방법을 제공
- 내부 구현 추상화
- 집합체 종류에 관계없이 사용 가능
- 모든 항목에 접근하는 작업을 컬렉션 객체가 아닌 반복자 객체가 처리(역할분리)

# 단일 역할 원칙

- 집합체에서 다 구현할 수도 있지만, 클래스에서 원래 클래스의 역할 외의 역할을 하게 되면 2가지 이유로 그 클래스가 변경 가능
  - 컬렉션이 어떤 이유로 바뀌겨 되면 그 클래스도 바뀌어야 함
  - 반복자 관련 기능이 바뀌었을 때에도 그 클래스가 바뀌어야 함
