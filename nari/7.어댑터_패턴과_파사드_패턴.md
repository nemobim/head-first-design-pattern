# Intro

- 어댑터 패턴: 실제와 다른 인터페이스를 가진 것처럼 보이도록 객체를 감싸는 것
- 파사드 패턴: 객체를 감싸서 인터페이스를 단순화 하는 패턴

# 어댑터 살펴보기

- 어댑터의 역할: 국산 전원 플러그를 영국식 소켓에 꽂을 수 있게 함
- 객체지향 어댑터의 역할: 어떤 인터페이스를 클라이언트에서 요구하는 형태로 적응시키는 역할

# 어댑터 사용 방법 알아보기

- Duck 객체가 모자라서 Turkey 객체를 대신 사용해야 하는 상황
- 어댑터를 만들어서 Turkey -> 어댑터 -> Duck 이 되도록 함

# 오리 어댑터 테스트

```java
public class DuckTestDrive {
```

# 어댑터 패턴 알아보기

## 클라이언트에서 어댑터를 사용하는 방법

- 클라이언트에서 타깃 인터페이스로 메소드를 호출해서 어댑터에 요청을 보냄.
- 어댑터는 이미 어댑티 인터페이스로 그 요청을 어댑터에 관한 메소드 호출로 변환
- 클라이언트는 호출 결과를 받긴 하지만 중간에 어댑터가 있다는 사실을 모름

# 어댑터 패턴의 정의

- 어댑터 패턴: 특정 클래스 인터페이스를 클라이언트에서 요구하는 다른 인터페이스로 변환
- 인터페이스가 호출되지 않아 같이 쓸 수 없었던 클래스를 사용할 수 있게 도와줌

# 객체 어댑터와 클래스 어댑터

- 어댑터의 종류 : 객체 어댑터 & 클래스 어댑터
- 근데 클래스 어댑터 패턴을 쓰려면 다중 상속이 필요한데 자바는 다중 상속이 안되어서 언급안했음

# 실전 적용 어댑터 패턴

# 퍼사드 작동 원리 알아보기

1. 홈 시어터 시스템용 퍼사드를 만들기 watchMovie와 같이 몇가지 간단한 메소드만 들어있는 HomeTheaterFacade 클래스 생성 필요
2. 퍼사드 클래스는 홈 시어터 구성 요소를 하나의 서브시스템으로 간주하고, watchMovie() 메소드는 서브시스템의 메소드를 호출해서 필요한 작업을 처리
3. 클라이언트 코드는 서브시스템이 아닌 홈시어터 퍼사드에 있는 메소드를 호출

# 퍼사드 패턴의 정의

- 퍼사드 패턴: 서브시스템에 있는 일련의 인터페이스를 통합 인터페이스로 묶어줌. 고수준 인터페이스도 정의하므로 서브시스템을 더 편하게 사용 가능

# 최소 지식 원칙

- 객체 사이의 상호작용은 될 수 있으면 아주 가까운 '친구' 사이에서만 허용

# 친구를 만들지 않고 다른 객체에 영향력 행사하기

```java
public float getTemp(){
  // 원칙을 따르지 않는 경우
  Thermometer thermometer = station.getThermometer();
  return thermometer.getTemperature();

}
```

```java
// 원칙을 따르는 경우
public float getTemp(){
  return station.getTemperature();

}
```
